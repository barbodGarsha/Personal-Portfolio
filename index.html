My Workflow

Analysis

The first thing I do when approaching a new project is asking questions to determine the Goals, the Obstacles, and ways to get over those Obstacles. While trying to find the Solutions to your Problems I decide which technologies would work the best for our project.

Planning

When it comes to planning Scrum Board is my favorite. I use it to get a visual representation of the ongoing project, what is done, and what needs to be done.

Design

I start with Basic Sketching on a notebook and I give my ideas life in Adobe XD. After I feel the base structure of the project is working I move on to finding a color palette and my favorite stage of designing, adding the little details that make the project unique and even more fun to look at.

One thing I always do while going through this process is keeping in mind what technologies I decided on using and if my design is Compatible with them.

Development

Finally I start building the project with the decided technologies. I also establish if I want to use a specific Design Pattern, if its components need to be modular and etc.

My approach to each project will be different and based on the size of the project, the timetable, and the needs of the client.

Testing

Of course there is a Testing stage to make sure of the quality of the Work. Depending on the project my way of testing changes. For example for a long term project I would rather go with Test Driven Development and writing my Tests with React Testing library would be the first thing I do before I actually start building the project.
